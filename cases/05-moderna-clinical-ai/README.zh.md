## 业务问题

Moderna 的工作覆盖科学研究、临床开发、制造、法务、监管和商业部门。通用助手可以处理日常任务，但临床和监管工作需要可追溯证据与明确责任。全公司推广也不能只靠分发账号，员工需要训练、内部方法和围绕专业任务构建助手的能力。

## 交付过程

Moderna 从 2023 年初开始与 OpenAI 合作，先基于 OpenAI API 构建内部 mChat，随后把 ChatGPT Enterprise 推广到多个部门。推广同时配有访谈、培训、办公时间、内部论坛、管理层参与和生成式 AI 推广者网络。

员工为法务、研究、制造、营销和销售创建任务型 GPT。公开案例中的 Dose ID 会支持临床研究团队评估疫苗剂量，按照标准准则处理信息，引用来源、解释理由并生成可视化，最终决定仍由临床团队作出。

~~~mermaid

  A[获批企业数据与文档] --> B[mChat、ChatGPT Enterprise 与自定义 GPT]
  B --> C[领域分析或草稿]
  C --> D[来源、理由与可视化]
  D --> E[科学家、临床、法务或业务复核]
  E --> F[获批决定或文档]
  F --> G[培训、反馈与可复用 GPT 模式]
  G --> B
~~~

## 采用与人工控制

- 在更大范围企业推广前，mChat 已有超过 80% 的员工使用。
- 推广者和办公时间帮助团队从提示词练习走向工作助手。
- 管理层直接参与，不把采用问题只交给 IT。
- 高风险输出被定位为决策支持，领域专家保留批准权。
- GPT 分布在多个部门，而不是集中在单一创新团队。

领域专家不只是最终用户，他们还参与定义问题、构建助手、复核结果和推广有效方法。

## 公开结果

Moderna 与 OpenAI 披露，ChatGPT Enterprise 部署后的前几个月，公司创建了 750 多个自定义 GPT。Moderna 的 2025 年股东信随后表示，公司已经创建 4,000 多个 GPT，并把合作用于方案撰写、监管文档、试验设计、数据分析、预测和内部流程。

GPT 数量增长证明采用和实验活跃，但不能证明每个 GPT 都在持续使用、安全或创造价值。Dose ID 这类明确说明决策边界与来源要求的工作流，证据价值更高。

## 为什么属于高置信度 AI FDE 案例

项目把技术部署、领域责任和变更管理连接起来，也为高风险 AI 划定合理边界：系统负责收集、解释和可视化信息，合格人员负责临床判断。可复用资产不仅是模型配置，还包括培训、推广者网络和基于来源的助手模式。

## 尚未证明的部分

- Dose ID 的评估集、错误率、验证阈值和对试验结果的影响没有公开。
- GPT 数量不能代表整体质量。
- 生产力和科学影响大多是定性描述。
- 临床支持案例不能理解为自主诊断或治疗选择。
