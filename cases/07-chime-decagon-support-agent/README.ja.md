## 業務上の課題

Chimeは、正確性、信頼性、セキュリティ、会員中心の体験を維持しながらサポート規模を拡大する必要がありました。対象はチャットと電話の両方で、問い合わせには機密性の高いアカウント情報や、カード交換、入金状況の確認といった操作が含まれます。

チャットと音声に別々の自動化システムを使うと、知識が重複し、方針が不一致になり、フィードバックも分断されます。Chimeが必要としたのは、構造化された依頼を解決しながら、難しい案件と中核の事業ロジックを人と技術チームが管理できる一つの運用モデルでした。

## 導入の進め方

Chimeは応答の正確性、自動化率、運用効率、サポートNPSを使って複数のパートナーを比較し、チャットと音声の両方でDecagonを選びました。

導入された設計は、チャネルをまたいで知識と挙動の層を共有します。会話は共通のフィードバックループへ入り、チャットで得た学びを音声へ、音声で得た学びをチャットへ反映できます。顧客体験チームは自然言語のAgent Operating Proceduresで挙動を調整し、Chimeの技術チームは中核ロジックと会員体験の統制を維持します。

Decagonによると、同社のチームはChimeの製品、運用チームと直接協働しました。短い周期でフィードバックを受け、用途と挙動を変更し、会話から得た知見をサポートだけでなく製品改善にも利用しました。

## システムと統制の設計

| 課題 | 公開資料で説明された対応 |
| --- | --- |
| チャネルの一貫性 | チャットと音声が知識、会話文脈、改善シグナルを共有する。 |
| 事業ルール | 運用チームはAgent Operating Proceduresで挙動を記述し、技術チームが中核ロジックを管理する。 |
| 人への移行 | 自動化範囲外の案件を文脈付きで人のサポートへ渡す。 |
| 品質 | 自動解決率だけでなく、正確性、解決、効率、会員満足度を評価する。 |
| 継続的改善 | Chimeの製品、運用チームとDecagonが用途とエージェント挙動を調整する。 |

## 公開された成果

DecagonのChime事例は、次の成果を報告しています。

- チャットで70%を超える解決率。
- 音声で約70%の解決率。
- 最初の2週間で数十万件のメッセージを自動処理。
- 月間100万件を超える通話を処理し、信頼性上の問題は報告されなかった。
- 比較対象の他社より会員満足度が改善。

Decagonの別資料は、サポートコストが60%減り、会員満足度が2倍になったと述べています。しかし測定期間と方法が公開されていないため、独立検証済みの成果ではなく、顧客とベンダーが発表した数値として扱う必要があります。

## AI FDE事例として確信度が高い理由

規制対象の事業、複数チャネル、アカウント単位の文脈、業務操作、技術的ガードレール、継続的フィードバックを含みます。また、前線導入で必要な責任分担も明確です。顧客体験チームが挙動を定義し、エンジニアが強制可能な統制と統合を所有し、導入パートナーが事業方針を本番システムへ変換します。

## 未確認の点

- Chimeはこの導入について独立した技術報告書を公開していません。
- 解決率の定義、評価サンプル、エスカレーションの分布は非公開です。
- 信頼性上の問題がなかったという表現は顧客事例の記述であり、公開された可用性指標ではありません。
- コストと満足度は、他のサポートシステムと比較する前に詳細な測定条件が必要です。
