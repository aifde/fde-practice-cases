## 業務上の課題

BBVAは生成AIを点在する実験から全行能力へ変える一方、機密性の高い銀行情報が統制されていないツールへ流れることを防ぐ必要がありました。複数の国と業務をまたぎ、法務とセキュリティを満たし、単なる文章作成ではなく業務を改善する用途を選ぶ必要があります。

技術と組織の両方が対象でした。専門家へ安全な環境を提供し、アイデアを動く支援機能へ変え、価値の高い用途を審査し、個人の試作を全行で再利用できる仕組みへ移します。

## 導入の進め方

BBVAは2024年5月にChatGPT Enterpriseの配布を始めました。初期は約3,000人で、必須研修、多国間コミュニティ、フィードバック経路を用意し、法務、コンプライアンス、セキュリティ、技術チームを最初から参加させました。

社員は支店向け法務回答、企業資料からの信用分析、自由記述アンケートの顧客体験分析など、実務別のGPTを作りました。中央の利用促進チームとチャンピオンが、有効な方法の検証と共有を支えました。

協力はその後、顧客体験、リスク、運用、開発、社員支援の共同変革へ拡大しました。BBVAはインテリジェントエージェントを構築・統制する共通基盤と部品も整備しています。

~~~mermaid
flowchart LR
  A[事業チームと業務専門家] --> B[安全な企業AI環境]
  B --> C[業務別支援と試作]
  C --> D[法務、セキュリティ、コンプライアンス、品質審査]
  D --> E[承認済み業務への導入]
  E --> F[利用と事業フィードバック]
  F --> G[共通部品、研修、エージェント基盤]
  G --> B
~~~

## ガバナンスと利用促進

| 課題 | 公開された対応 |
| --- | --- |
| データ保護 | 企業アカウント、機密情報の指針、法務・セキュリティ・コンプライアンスの参加。 |
| 利用能力 | 必須研修、ワークショップ、チャンピオン、上級利用者、内部コミュニティ。 |
| 用途発見 | 法務、リスク、運用、顧客業務の専門家が直接構築。 |
| 再利用 | 社内GPT Storeと共通エージェント部品で成功例を共有。 |
| 拡大 | 全業務を一度に開放せず、段階的に対象を広げた。 |

## 公開された成果

- 2025年5月には11,000アカウントへ拡大し、対象者の83%が毎日利用、週平均2.8時間の自己申告による削減。
- 2025年12月には12万人を超える社員へ開放し、OpenAIとの共同開発を拡大。
- 2026年半ばには約10万人の継続利用者、2万件を超えるGPT作成、一部業務で最大80%の効率改善を報告。
- 2026年4月には8,000件を超える利用中の用途を報告し、その一部を戦略的な再利用候補とした。

数値は時期と対象者が異なり、単一の測定として合算できません。

## AI FDE事例として確信度が高い理由

多様な関係者の合意、安全な利用環境、研修、専門家による試作、業務審査、共通部品を一体で進めています。現場での利用結果が協力内容を変え続けており、ライセンス購入だけで終わっていません。

## 未確認の点

- 時間削減は自己申告で、職種と作業によって差があります。
- 利用数とGPT数は採用指標であり、顧客価値や財務価値ではありません。
- 公開資料は成功例が中心で、全体の失敗率や投資収益は不明です。
- 2025年末に発表した顧客向け計画は、実証済み機能と分ける必要があります。

## FDE職務能力との対応

- **R1. 現場に入る課題発見と信頼関係**
- **R2. スコープ設定と一貫したデリバリー責任**
- **R3. 自ら行う実装とシステム統合**
- **R4. AIワークフローと人による統制の設計**
- **R6. セキュリティ、ガバナンス、信頼性、コスト**
- **R7. 利用定着と測定可能な事業効果**
- **R8. 再利用可能な型と現場フィードバック**
