## 业务问题

律师起草和谈判合同时，需要从过去协议、沟通记录和客户文档中找到相关语言。关键词搜索可能漏掉语义相近的条款，不受约束的生成又可能给出难以验证的文字。法律 AI 必须帮助用户找到先例，并说明建议来自哪里。

Draftwise 的目标不是替代法律判断，而是减少重复搜索和起草工作，同时让客户经验与来源文档始终对律师可见。

## 交付过程

Draftwise 先访谈潜在客户，理解律师怎样搜索、起草和审阅协议，再通过 Azure AI Foundry 使用 Cohere 模型构建 Smart Draft 的检索增强生成系统。

- Cohere Embed 按语义表示合同和其他材料。
- Cohere Rerank 调整检索结果顺序，让最相关示例优先出现。
- Cohere Command 生成起草建议并引用来源文档。
- 微调让模型适应法律搜索和产品要求。

## 评估与控制

公开案例关注搜索结果质量，而不是通用模型榜单。这是正确的产品单位：如果检索没有找到有用条款，律师就无法复核。

可追溯引用是第二道控制。用户可以直接检查原协议，而不是因为文字看起来专业就相信生成结果。团队选择 Azure 部署，也部分因为创始人与客户已经信任这套企业环境。

完整法律生产系统还需要事务级权限、信息隔离、保留政策、保密特权和版本历史。公开案例解释了检索与生成，但没有披露整套律所治理模式。

## 公开结果

Cohere 与 Draftwise 披露：

- 引入经过微调的 Cohere 模型后，内部基准中的搜索结果质量提高 30%。
- 2025 年第一季度，模型 API 调用量增长到三倍。
- 律师能够更快找到过去协议中的相关条款和示例。

30% 是与系统直接相关的指标，但数据集、基线和统计方法没有公开。API 增长只能作为采用信号，不能证明法律质量或节省时间。

交付从领域访谈和专业工作流开始，架构组合生成、向量表示、重排、引用和安全云部署，并把律师放在最终决策点。模型选择只是整个实现中的一部分。
