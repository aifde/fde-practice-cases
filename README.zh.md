## “高置信度”的含义

一个案例只有在公开证据能够支持以下全部或绝大多数条件时才会纳入：

1. 有明确的客户或运营机构，也有明确的 AI 交付伙伴。
2. 有具体的 AI 工作流，而不是泛泛的合作公告。
3. 已进入生产、实际运营，或已经成为正式发布的企业产品。
4. 有集成、评估、治理、采用情况或可量化成果的证据。
5. 在可能的情况下交叉使用客户、供应商、监管文件、产品资料或独立报道。
6. 信息足以说明 AI FDE 需要发现什么、构建什么、如何测试、如何治理、如何持续改进。

高置信度不表示每个数字都经过独立审计。每个案例都会区分已确认的部署事实、参与方披露的成果，以及仍未证实的主张。

## 范围与独立性

- “FDE 式”描述一种交付方法：深入用户现场，连接生产系统，在真实工作流中构建和评估，管理风险，衡量结果并持续迭代。它不表示每个机构都采用了正式的 FDE 职称。
- 案例覆盖医疗、政府网络安全、银行、生命科学、客户支持、云基础设施、法律科技和企业应用。
- 每个案例使用 Mermaid 图解释系统与人工流程，不转载受版权保护的营销图片。
- 本项目是独立社区资料，与 OpenAI、Amazon、Anthropic、Cohere、Palantir、Decagon 或文中客户机构不存在隶属或背书关系。

## 贡献

请只提交有公开来源支持的生产案例。一个合格的新增案例应说明业务负责人、交付伙伴、真实工作流、接入系统、人工控制点、评估方式、运营成果和证据边界。
